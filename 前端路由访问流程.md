# 前端路由访问流程详解

## 📍 用户访问项目管理页面的完整流程

### 访问 URL：`http://localhost:5173/dashboard/projects`

---

## 🔄 完整流程（包含权限判断）

```
1. 用户访问：http://localhost:5173/dashboard/projects
    ↓
2. 路由守卫检查 token
    ↓
    ├─────────────────────┬──────────────────────┐
    ↓                     ↓                      ↓
3a. 有 token           3b. 没有 token        3c. 有 token 但访问登录页
    ↓                     ↓                      ↓
4a. 允许访问            4b. 重定向到/login      4c. 重定向到/dashboard
    ↓                     ↓                      ↓
5a. 加载布局            5b. 用户看到登录页      5c. 用户看到仪表盘
    ↓
6a. 加载 Projects.vue
    ↓
7a. 用户看到项目管理页面
```

---

## 📊 详细步骤说明

### 步骤 1：用户访问 URL
```
用户输入或点击链接：
http://localhost:5173/dashboard/projects
```

### 步骤 2：路由守卫检查 token
```javascript
// router/index.js 第 64-74 行
router.beforeEach((to, from, next) => {
  const token = localStorage.getItem('token') || sessionStorage.getItem('token')
  
  // 判断逻辑开始
  if (to.path !== '/login' && !token) {
    next('/login')
  } else if (to.path === '/login' && token) {
    next('/dashboard')
  } else {
    next()
  }
})
```

### 步骤 3：三种情况分支

#### 情况 A：有 token ✅
```
条件判断：
- to.path !== '/login' → true（访问的是 /dashboard/projects）
- !token → false（有 token）
- 进入 else 分支 → next()

结果：允许访问，继续加载页面
```

#### 情况 B：没有 token ❌
```
条件判断：
- to.path !== '/login' → true（访问的是 /dashboard/projects）
- !token → true（没有 token）
- 进入 if 分支 → next('/login')

结果：重定向到登录页
```

#### 情况 C：有 token 但访问登录页 ⚠️
```
条件判断：
- to.path === '/login' → true（访问的是 /login）
- token → true（有 token）
- 进入 else if 分支 → next('/dashboard')

结果：已登录用户不允许访问登录页，跳转到仪表盘
```

### 步骤 4-7：情况 A 的后续流程（有 token）

```
4. 允许访问
    ↓
5. 加载 DashboardLayout.vue 布局
   - 侧边栏导航
   - 顶部导航栏
   - 主内容区域
    ↓
6. 在布局中加载 Projects.vue 组件
   - 项目列表
   - 项目表格
   - 操作按钮
    ↓
7. 用户看到完整的项目管理页面
```

### 步骤 4-5：情况 B 的后续流程（没有 token）

```
4. 自动重定向到 /login
    ↓
5. 用户看到登录页面
   - 邮箱输入框
   - 密码输入框
   - 登录按钮
```

## 🎯 路由匹配过程

### 1. 路径匹配
```
访问路径：/dashboard/projects

匹配路由配置：
{
  path: '/dashboard',
  component: DashboardLayout,
  children: [
    {
      path: 'projects',    // ← 匹配到这里
      name: 'Projects',
      component: Projects
    }
  ]
}
```

### 2. 组件加载顺序
```
第一层：DashboardLayout.vue（父路由组件）
  └─ 第二层：Projects.vue（子路由组件）
```

### 3. 页面渲染
```html
<div id="app">
  <router-view>
    <!-- DashboardLayout.vue -->
    <div class="dashboard-layout">
      <sidebar />
      <header />
      <main>
        <router-view>
          <!-- Projects.vue -->
          <div class="projects-page">
            <!-- 项目列表内容 -->
          </div>
        </router-view>
      </main>
    </div>
  </router-view>
</div>
```

| /dashboard/projects | 无 token | 拒绝访问 | /login ❌ |
| /login | 有 token | 已登录 | /dashboard ⚠️ |

| /dashboard | 有 token | 允许访问 | /dashboard ✅ |

## 🔐 Token 存储位置

```javascript
// 路由守卫检查这两个地方
const token = localStorage.getItem('token') || sessionStorage.getItem('token')
```

**localStorage：**
- 持久化存储
- 关闭浏览器后仍然存在
- 用于"记住我"功能

**sessionStorage：**
- 会话级存储
- 关闭浏览器后清除
- 用于临时登录

## 💡 实际应用场景

### 场景 1：首次访问（未登录）
```
用户打开浏览器 → 访问 /dashboard/projects
→ 没有 token → 重定向到 /login
→ 用户登录 → 获取 token → 跳转到 /dashboard
```

### 场景 2：已登录访问
```
用户已登录 → 访问 /dashboard/projects
→ 有 token → 允许访问
→ 加载项目管理页面
```

### 场景 3：登录后访问登录页
```
用户已登录 → 手动访问 /login
→ 有 token → 不允许访问登录页
→ 自动跳转到 /dashboard
```

### 场景 4：Token 过期
```
用户刷新页面 → token 过期或无效
→ 检查 token → 无有效 token
→ 重定向到 /login
→ 用户需要重新登录
```
