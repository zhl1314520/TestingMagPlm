# MySQL 数据库优化方案

> TestingMagPlm 测试管理平台数据库优化建议

---

## 📋 当前问题

| 问题 | 影响 |
|------|------|
| 索引缺失 | 登录、筛选查询慢（无索引） |
| 外键缺失 | 数据不一致风险 |
| 字段不全 | 缺少更新时间、软删除等 |
| 枚举无约束 | status/priority 用 VARCHAR |

---

## 🚀 优化方案

### 1. 用户表 (users)

```sql
CREATE TABLE users (
    id INT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL,
    password VARCHAR(255) NOT NULL,
    role ENUM('admin', 'manager', 'tester', 'user') NOT NULL DEFAULT 'user',
    email VARCHAR(100) NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at DATETIME DEFAULT NULL,
    
    UNIQUE KEY uk_username (username),
    UNIQUE KEY uk_email (email),
    INDEX idx_role (role),
    INDEX idx_deleted_at (deleted_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**优化点：**
- ✅ `email` 添加唯一索引（登录查询）
- ✅ `role` 使用 ENUM 约束
- ✅ 添加 `updated_at`、`deleted_at`

---

### 2. 项目表 (projects)

```sql
CREATE TABLE projects (
    id INT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    owner_id INT UNSIGNED NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at DATETIME DEFAULT NULL,
    
    INDEX idx_owner_id (owner_id),
    INDEX idx_deleted_at (deleted_at),
    
    FOREIGN KEY (owner_id) REFERENCES users(id) ON DELETE RESTRICT
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**优化点：**
- ✅ 添加外键约束
- ✅ 添加 `updated_at`、`deleted_at`

---

### 3. 项目成员表 (project_members)

```sql
CREATE TABLE project_members (
    id INT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    project_id INT UNSIGNED NOT NULL,
    user_id INT UNSIGNED NOT NULL,
    role ENUM('manager', 'tester', 'viewer') DEFAULT 'tester',
    joined_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    UNIQUE KEY uk_project_user (project_id, user_id),
    INDEX idx_user_id (user_id),
    
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**优化点：**
- ✅ 添加唯一约束（防止重复添加成员）
- ✅ 添加外键级联删除

---

### 4. 测试用例表 (test_cases)

```sql
CREATE TABLE test_cases (
    id INT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    project_id INT UNSIGNED NOT NULL,
    module VARCHAR(100) NOT NULL,
    title VARCHAR(255) NOT NULL,
    steps TEXT NOT NULL,
    expected TEXT NOT NULL,
    status ENUM('draft', 'active', 'deprecated', 'blocked') DEFAULT 'active',
    priority ENUM('p0', 'p1', 'p2', 'p3') DEFAULT 'p2',
    created_by INT UNSIGNED DEFAULT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at DATETIME DEFAULT NULL,
    
    INDEX idx_project_id (project_id),
    INDEX idx_module (module),
    INDEX idx_status (status),
    INDEX idx_project_module_status (project_id, module, status),
    
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
    FOREIGN KEY (created_by) REFERENCES users(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**优化点：**
- ✅ 复合索引 `(project_id, module, status)` 优化筛选
- ✅ 添加 `priority`、`created_by`

### 5. 执行记录表 (executions)

```sql
CREATE TABLE executions (
    id INT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    project_id INT UNSIGNED NOT NULL,
    name VARCHAR(200),
    type ENUM('manual', 'auto') DEFAULT 'manual',
    status ENUM('pending', 'running', 'completed', 'failed', 'cancelled') DEFAULT 'pending',
    result TEXT,
    total_cases INT UNSIGNED DEFAULT 0,
    passed_cases INT UNSIGNED DEFAULT 0,
    failed_cases INT UNSIGNED DEFAULT 0,
    pass_rate DECIMAL(5,2) DEFAULT 0.00,
    executed_by INT UNSIGNED DEFAULT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_project_id (project_id),
    INDEX idx_status (status),
    INDEX idx_project_status (project_id, status),
    
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
    FOREIGN KEY (executed_by) REFERENCES users(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**优化点：**
- ✅ 添加统计字段（用例数、通过数等）
- ✅ 复合索引优化查询

### 6. 缺陷表 (bugs)

```sql
CREATE TABLE bugs (
    id INT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    project_id INT UNSIGNED NOT NULL,
    testcase_id INT UNSIGNED DEFAULT NULL,
    title VARCHAR(255) NOT NULL,
    description TEXT NOT NULL,
    status ENUM('new', 'open', 'in_progress', 'fixed', 'verified', 'closed') DEFAULT 'new',
    priority ENUM('critical', 'high', 'medium', 'low') DEFAULT 'medium',
    assignee_id INT UNSIGNED DEFAULT NULL,
    reporter_id INT UNSIGNED DEFAULT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    deleted_at DATETIME DEFAULT NULL,
    
    INDEX idx_project_id (project_id),
    INDEX idx_status (status),
    INDEX idx_priority (priority),
    INDEX idx_assignee_id (assignee_id),
    INDEX idx_project_status_priority (project_id, status, priority),
    
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
    FOREIGN KEY (testcase_id) REFERENCES test_cases(id) ON DELETE SET NULL,
    FOREIGN KEY (assignee_id) REFERENCES users(id) ON DELETE SET NULL,
    FOREIGN KEY (reporter_id) REFERENCES users(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**优化点：**
- ✅ 复合索引 `(project_id, status, priority)` 优化组合查询
- ✅ 添加 `reporter_id`、`updated_at`

### 7. 报告表 (reports)

```sql
CREATE TABLE reports (
    id INT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    project_id INT UNSIGNED NOT NULL,
    execution_id INT UNSIGNED DEFAULT NULL,
    title VARCHAR(200),
    pass_rate DECIMAL(5,2) NOT NULL DEFAULT 0.00,
    fail_rate DECIMAL(5,2) NOT NULL DEFAULT 0.00,
    total_cases INT UNSIGNED DEFAULT 0,
    passed_cases INT UNSIGNED DEFAULT 0,
    failed_cases INT UNSIGNED DEFAULT 0,
    created_by INT UNSIGNED DEFAULT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_project_id (project_id),
    INDEX idx_created_at (created_at),
    INDEX idx_project_created (project_id, created_at),
    
    FOREIGN KEY (project_id) REFERENCES projects(id) ON DELETE CASCADE,
    FOREIGN KEY (execution_id) REFERENCES executions(id) ON DELETE SET NULL,
    FOREIGN KEY (created_by) REFERENCES users(id) ON DELETE SET NULL
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

**优化点：**
- ✅ 复合索引 `(project_id, created_at)` 优化趋势查询
- ✅ 添加详细统计字段

### 8. 系统名称表 (system_name)

```sql
CREATE TABLE system_name (
    id INT UNSIGNED PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    description VARCHAR(255),
    status ENUM('active', 'inactive') DEFAULT 'active',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    UNIQUE KEY uk_name (name),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## ⚠️ 实施步骤

### 1. 备份

```bash
mysqldump -u root -p tmp_db > backup_$(date +%Y%m%d).sql
```

### 2. 添加索引（无风险）

```sql
ALTER TABLE users ADD UNIQUE INDEX idx_email (email);
ALTER TABLE test_cases ADD INDEX idx_project_module_status (project_id, module, status);
ALTER TABLE bugs ADD INDEX idx_project_status_priority (project_id, status, priority);
ALTER TABLE reports ADD INDEX idx_project_created (project_id, created_at);
```

### 3. 添加字段

```sql
ALTER TABLE users ADD COLUMN updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP;
ALTER TABLE users ADD COLUMN deleted_at DATETIME DEFAULT NULL;
ALTER TABLE users ADD COLUMN status ENUM('active', 'inactive', 'locked') DEFAULT 'active' AFTER role;
```

### 4. 添加外键（需验证数据）

```sql
ALTER TABLE projects ADD CONSTRAINT fk_projects_owner FOREIGN KEY (owner_id) REFERENCES users(id);
ALTER TABLE test_cases ADD CONSTRAINT fk_test_cases_project FOREIGN KEY (project_id) REFERENCES projects(id);
```
